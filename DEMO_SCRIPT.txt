[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at the workflow architecture that powers this. Every student query starts at the START node and is immediately sent to the security_checkpoint. If the checkpoint passes, control is handed to our main coordinator, the edu_orchestrator agent. Using AgentTools, the orchestrator delegates tasks to our three specialized sub-agents: the study_planner, the quiz_generator, and the explanation_helper. These sub-agents are wired with an MCPToolset that communicates with our custom local MCP server to fetch learning tips, create mnemonics, or simplify equations. After compiling the response, the orchestrator routes to the feedback_requester node, which triggers a Human-in-the-Loop pause. The student's feedback is then routed back to the security checkpoint, creating a secure, iterative loop until the session is completed and final_output is returned.

[3:00 — SECURITY & MCP]
This security checkpoint is extremely robust. It scrubs personally identifiable info like emails and phone numbers, checks for prompt injection, enforces academic integrity to prevent homework cheating, and restricts sessions to 3 revisions to protect API limits. Meanwhile, our custom MCP server runs in the background using stdio transport, offering targeted local helper functions to the agents.
